Ceros Element Naming Conventions

Internal Governance for SDK-Compatible Templates
Version: 1.0
Date: 2025-06-11

1. Purpose

  • Enable robust interaction via Ceros SDK
  • Facilitate modular SDK code
  • Support Studio-based edits without developer involvement
  • Improve readability, maintainability, and collaboration

2. Scope

  • Custom JavaScript via the Ceros SDK
  • Embedded JSON configuration
  • Event tracking or analytics
  • Reusable templates

3. Naming Syntax

AttributeConvention
Element NameLowercase kebab-case (carousel-slide-1)
Prefix Conventiontype-role-index (e.g., button-cta-1)
DelimitersUse - for word separation
Dynamic Element FlagsAppend --interactive, --trackable, --conditional as needed

4. Element Type Prefixes

PrefixMeaning
img-Static or dynamic image
txt-Text block
btn-Button or interactive element
grp-Group container
vid-Video element
bg-Background asset
nav-Navigation container or trigger

5. Role-Based Suffixes

SuffixMeaning
-ctaCall-to-action
-containerParent group or holder
-slideCarousel or slideshow frame
-triggerBehavior activation
-targetDynamic content target
-stateXVisual state indicator (e.g., state1, state2)

6. Dynamic Behavior Flags

  • --interactive: JS binds interactivity to this element
  • --trackable: Tracked for analytics
  • --auto-hide: Hidden on load unless triggered
  • --conditional: Appears only under logic conditions

Example: btn-download-1--interactive--trackable

7. Metadata Tagging (Alt Text / Notes Usage)

FieldUse Case
NotesJSON-like key-value pairs for config ({"event":"cta-click", "target":"modal-1"})
Alt TextDescriptive or fallback content
Object NamePrimary selector for SDK behavior

8. ID Uniqueness & Version Control

  • Every interactive element must have a unique name
  • Append experience-specific namespaces if needed

9. Reserved Terms

Avoid using the following terms:

  • header, footer, nav
  • main, content
  • test, temp, draft

Appendix C: Linter Tool (optional JS validator)

function validateElementNames(experience) {
  const elements = experience.getAllElements();
  elements.forEach(el => {
    if (!/^[a-z]+-[a-z]+(-[0-9]+)?(--[a-z]+)*$/.test(el.name)) {
      console.warn("Naming violation:", el.name);
    }
  });
}
