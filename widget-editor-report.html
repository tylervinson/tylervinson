Editor Modes
Content Editor

Content Editor Mode

Detected by URL pattern and the presence of scContentIframeId0. Reads and writes widget HTML via the Sitecore SSC REST API, then automates the "Edit HTML → Accept" button flow to commit changes into the CMS.

  • Reads field HTML via /sitecore/api/ssc/aggregate/content/Items/Fields
  • Auto-detects current item ID from Quick Info input
  • Monitors tree navigation — rescans automatically on item switch
  • Writes back through the native HTML editor dialog
  • Supports all Rich Text fields on the active item
Experience Editor

Experience Editor Mode

Detected by scanning DOM across all frames for registered widget CSS classes. Reads from and writes directly to live page elements. Changes are immediately visible in the WYSIWYG canvas.

  • Scans all frames, deduplicates widget instances
  • Live DOM reads and writes — no API calls required
  • Fires Sitecore bridge events after every change
  • Calls PageEditor.setModified() and ChromeManager.resetChromes()
  • Widget picker shown when multiple widgets are found
Managed Widgets

Animated Cards

.card-row
Standard Repeat

A three-card statistics widget where each card displays an animated number counter and a label. Numbers count up from zero to their target value when the widget scrolls into view.

The extension manages the numeric target value and the label text on each card. When a number is written, it is also synced to the data-target attribute which drives the JavaScript counter animation — ensuring the displayed number and the animation target stay in sync.

Insurance Plan Comparison Tool

.insurance-comparison-tool
Standard Repeat + Standalone

A side-by-side insurance plan comparison widget with up to 4 plans. Each plan has a selector button at the top and a detailed comparison grid below. Selecting a plan highlights its column across all comparison rows.

The extension edits both the plan selector buttons (name, coverage label, price) and the corresponding cells in the comparison grid — the gridSelector / gridCellSelector field type handles the grid cell writes. Numeric fields like price and deductible are also synced to data attributes used by the grid's JavaScript.

A standalone section heading field sits above the repeat group and controls the widget's title.

Zoom Meetings

.zoom-meetings
Standard Repeat

A scheduled meetings list where each row displays a date block (month, day, time), a title, a description, and a "Join Meeting" link. Meetings with a data-end date in the past are automatically hidden from view via a CSS collapse animation.

The extension can edit all fields on each meeting row including the join URL, which is read from and written to the href attribute of the link element rather than its text content. Meetings can be added (clones the last row with blank fields) or deleted (with a guard preventing removal of the last remaining item).

Image Grid Widget

.image-grid-widget
Special Type: image-grid

A flexible image grid with multiple layout presets. The extension renders a completely custom editing UI for this widget — instead of the standard field form, it shows a layout picker dropdown and a draggable list of images.

Layouts are defined in the registry and map to CSS snippets that are injected into the widget's internal <style> block on apply. The layout picker is context-aware, showing only layouts compatible with the current number of visible images.

Each image row has a drag handle for reordering and a Show/Hide toggle. On apply, the extension rewrites data-order, data-hidden, and CSS order on each image element, and replaces the style block with the selected layout CSS.

Core Capabilities
＋

Add Items

Clones the last repeat item and clears all field values, inserting it immediately after. Panel rescans and restores the active widget on completion.

✕

Delete Items

Removes any repeat item by index. Guarded — the last remaining item cannot be deleted. Panel rescans after deletion.

↻

Registry Refresh

Refresh button in the panel footer re-fetches the live registry on demand, without reopening or reinstalling the extension.

!

Update Notifications

Background service checks for registry version changes every 4 hours. A badge and in-panel banner appear when a new version is detected.

⌗

Attr Field Type

Fields marked with attr read from and write to HTML attributes (e.g. href) rather than text content — enabling link URL editing.

⇄

Data Attr Sync

Fields marked with syncDataAttr write their value to both the DOM element and a specified data attribute — keeping JS-driven components in sync.

Registry Update Flow

Adding new widgets or editing existing fields requires only a registry JSON push — no extension reinstall needed. An extension reinstall is only required when the core JS files change.

Notes & Known Constraints
⚠
API key is hardcoded in content-script.js as a GUID. It's not a secret credential per se, but it's bundled in the extension ZIP and visible to anyone who unpacks it.
⚠
Add/Delete (CLONE_ITEM / DELETE_ITEM) are only explicitly handled in the Experience Editor branch of the message handler. In Content Editor mode, these messages fall through to the EE path — behavior may differ in CE depending on page context.
⚠
Image Grid editing is likely only fully functional in Experience Editor mode. In Content Editor mode, UPDATE_WIDGET_DATA goes through buildUpdatedHtml which does not have image-grid awareness — the dedicated buildImageGridHtml function would not be called.
ℹ
No bundled registry file — the onInstalled handler attempts to load a widget-registry.json from the extension package, but no such file is included in the ZIP. The panel relies entirely on the remote registry URL from first open.
ℹ
Retry on scan — the panel retries widget scanning up to 6 times at 600ms intervals to handle pages where iframes or content may not be ready immediately on open.
