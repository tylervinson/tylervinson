Traditional Chinese (Taiwan) — Language Label Guidance


1 · Problem summary

Incorrect use of "中国人 / 中國人"

The label 中国人 / 中國人 is being used to represent Traditional Chinese (Taiwan) in a language selector. This is incorrect because it translates to "Chinese person" — a national or ethnic identity — not a language name. It also introduces political and cultural sensitivity, particularly for users in Taiwan.

2 · TL;DR

Summary

Using "中国人 / 中國人" as a language label is linguistically incorrect, not aligned with industry standards (W3C, CLDR, Google, Apple, Microsoft), and likely to cause user friction — especially for Taiwan users.

Replacing it with 中文（繁體） aligns with global standards and improves user experience without ambiguity or political sensitivity.

3 · Industry standard

Language selectors must use language names, not identities

Per W3C Internationalization guidance, language pickers should display the name of the language in that language. Labels must represent languages, not countries or people.

Standard practice for Chinese variants:

Simplified Chinese → 中文（简体） Traditional Chinese → 中文（繁體）

Optional regional variants: 中文（台灣）  中文（香港）

Key principle: use language name + script and/or region. Avoid national identity terms.

4 · Google standard

What Google Translate uses

Google Translate UI uses 中文（繁體） as the default label for Traditional Chinese, and sometimes 中文（台灣） for regional specificity.

Important: the Google Translate API only returns codes (e.g. zh-TW, zh-Hant) — it does not provide display labels. UI labels follow CLDR localization standards.

What Google avoids:

中国人 / 中國人 中国 / 中國

7 · Rationale

Why this matters

  • Accuracy — "中国人" refers to people, not a language
  • Consistency — aligns with W3C, CLDR, and all major platforms
  • User expectations — Taiwan users expect Traditional Chinese labeling
  • Neutrality — avoids political and cultural sensitivity
