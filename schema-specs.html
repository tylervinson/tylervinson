Architecture

System Components

How the generator scrapes, generates, validates, and stores schemas.

Web Scraper

Fetches and parses live web pages using CORS proxies and DOM parsing.

The scraper uses a fallback chain of CORS proxies to handle cross-origin restrictions. When direct access fails, it automatically tries secondary proxies including Cloudflare Worker, AllOrigins, CorsAnywhere, and others.

Once the HTML is retrieved, the scraper parses it with DOMParser to extract metadata including:

  • Page title, description, and canonical URL
  • Open Graph and Twitter Card metadata
  • Structured data already present (microdata, JSON-LD)
  • Images and image dimensions
  • Breadcrumb trails
  • Author and publisher information

Schema Generator

Transforms extracted metadata into valid JSON-LD schemas.

The generator creates structured data following schema.org standards. It intelligently maps extracted fields to appropriate schema types (WebPage, Organization, Article, Product, etc.) and fills in all required and recommended properties.

The generator also:

  • Validates all URLs and formats
  • Generates unique @id values for each entity
  • Creates proper entity relationships (isPartOf, mainEntity, etc.)
  • Normalizes language codes and geographic regions

Validation Engine

Checks schemas against schema.org standards and assigns quality scores.

Every generated schema is validated for completeness and correctness. The validation engine checks:

  • Required properties are present
  • Property types match schema.org specifications
  • URLs are properly formatted and accessible
  • No deprecated properties are used

Quality is scored from 0-100 based on completeness of optional fields and presence of recommended properties.

GitHub Integration

Stores schemas and maintains version history in a GitHub repository.

Schemas are persisted to GitHub for version control and archival. Each schema is stored as a separate JSON file with the format schemas/[domain]/[path].json.

The integration supports:

  • Automatic commit messages with URL and timestamp
  • Branch selection (default: main)
  • Public read access via GitHub Raw API
  • Private write access using personal access tokens
  • Full version history and diff comparison
API Reference

Endpoints & Methods

Core functions and their parameters.

scrapePage(url, corsProxy)

Fetches and parses a web page, returning structured metadata.

Parameters:

Parameter Type Description
url string The full URL to scrape
corsProxy string Optional. CORS proxy to use if direct request fails

Returns:

Object containing html, title, description, image, author, and other extracted metadata.

generateSchemaFromPageData(pageData, siteConfig)

Converts extracted page data into a complete JSON-LD schema.

Parameters:

Parameter Type Description
pageData object Extracted metadata from scrapePage()
siteConfig object Site-wide configuration from the first page (organization, logo, etc.)

Returns:

Complete JSON-LD schema object following schema.org standards.

validateSchema(schema)

Validates a schema and returns a quality score.

Parameters:

Parameter Type Description
schema object JSON-LD schema to validate

Returns:

Field Type Description
isValid boolean Whether schema meets minimum requirements
score number Quality score 0-100
errors array List of validation errors (if any)
warnings array List of warnings about missing recommended fields

saveSchemaToGitHub(url, schema)

Persists a schema to GitHub with automatic version history.

Parameters:

Parameter Type Description
url string The original source URL (used for path generation)
schema string JSON-LD schema as JSON string

Requires:

GitHub configuration with personal access token (for write access).

Required Properties

Minimum fields that must be present in any valid schema.

Property Type Description
@context string Must be "https://schema.org"
@type string Schema type (WebPage, Article, Product, etc.)
@id string Unique identifier with fragment (e.g., #webpage)
name string Page or content title
url string Canonical URL of the page
Integration

Using Schemas

How to implement generated schemas in your website.

HTML Implementation

Add the generated schema to your page using a script tag.

Copy the schema JSON from the generator and paste it into your HTML page within a <script type="application/ld+json"> tag in the page <head> or <body>:

<head>
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "WebPage",
    ...
  }
  </script>
</head>

Testing & Validation

Verify your schema is correct using Google's tools.

After implementing a schema, test it with:

  • Google Rich Results Test: https://search.google.com/test/rich-results
  • Schema.org Validator: https://validator.schema.org/
  • JSON-LD Playground: https://json-ld.org/playground/

These tools will verify that your schema is valid and identify any missing or incorrect properties.

SEO Benefits

How structured data improves search visibility.

Proper JSON-LD schemas enable search engines to:

  • Display rich snippets in search results (ratings, prices, availability)
  • Understand page content and relationships
  • Provide accurate information in Knowledge Panels
  • Improve click-through rates with enhanced appearance
  • Support voice search and smart assistants

Quality schemas with all recommended properties provide the most SEO benefit. The generator's quality score helps identify which properties to prioritize.

Troubleshooting

Common Issues

Solutions for typical problems and errors.

Schema generation fails with CORS error.

The page may have strict CORS restrictions. The generator automatically tries multiple CORS proxies. If all fail, check that the URL is correct and the page is publicly accessible.

Generated schema is missing important fields.

The scraper extracts metadata from page HTML and meta tags. If fields are missing, the page may not contain that information. You can manually add missing properties to the JSON before using it.

Quality score is low (below 60).

Add more optional recommended properties to improve the score. Common missing fields include: image dimensions, language code, publisher information, author details, and contact information. The validator tool will highlight which fields are missing.

URL path issues in GitHub storage.

Schemas are stored with paths based on domain and page URL. Special characters are sanitized. Ensure your GitHub token has write permission to the repository and the branch exists.

Bulk processing hangs or times out. Large batches (100+ URLs) may take several minutes. Each page requires scraping, parsing, and validation. For very large jobs, process URLs in smaller batches of 20-50 at a time.
