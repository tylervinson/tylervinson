The Dashboard

What opens when you click the icon

When the sidebar opens on a Sitecore Content Editor page, the first thing you see is the dashboard — not the widget editor. The dashboard is a lightweight content management hub.

Search

Full-text Sitecore content search from the sidebar

Type in the search bar at the top of the dashboard to search across Sitecore content. Results appear as clickable items — click any result to navigate the Content Editor to that item directly.

Recently Viewed

Items you've navigated to during this session

The sidebar records each item you've opened and displays the most recent 10 in the dashboard. Click any item to navigate back to it in the Content Editor. Recents persist across sessions using Chrome's local storage.

Checked Out

All items currently locked by any user

The "Checked Out" panel fetches all items currently locked in Sitecore and lists them by item name and path. This is useful for seeing at a glance what's in progress across the team without navigating to each item individually.

Editing Widgets

From detection to save

When the sidebar detects a registered widget on the current Sitecore item, it switches from the dashboard to the widget editor view automatically.

1

Navigate to the component in the Content Editor

In Sitecore's Content Editor, select the item in the tree that contains the widget in one of its Rich Text fields. The Quick Info panel on the right will show the item name, ID, and path — the extension reads the item ID from there automatically.

The extension works in the Content Editor only. It does not support Experience Editor.
2

Open the sidebar

Click the extension icon in the Chrome toolbar. The sidebar opens and immediately fetches the latest registry from the server, then scans all Rich Text fields on the current item via the SSC API.

If a registered widget is found, the editor form loads. If none is found, a "no widgets detected" message is shown. If you've navigated to a different item since the last scan, click the refresh button (↻) in the sidebar footer.

3

Edit fields in the form

The widget editor form is organized by the registry definition:

  • Standalone fields — appear at the top. These are single non-repeating elements like headings or subheadings.
  • Repeat groups — each repeating item (plan, card, meeting) gets its own collapsible labeled group. Groups are named automatically: "Plan 1", "Card 2", etc. If a groupLabelField is set, the group header shows the current value of that field live as you type.
  • Rich text fields — open a Quill editor inline. Supports bold, italic, links, ordered and unordered lists.
  • Date/time fields — render a native datetime-local picker.
4

Click Apply Changes

The extension silently automates the WYSIWYG HTML editor: it fetches the current field HTML via the SSC API, applies all edits using the registry selectors, serializes the result, then opens the "Edit HTML" dialog, pastes the updated markup, and accepts. The entire cycle completes in under a second with no visible modal.

Data attributes that require syncing (like data-target on the Animated Cards widget) are updated automatically alongside the visible text content.
5

Save in Sitecore

Click Save in the Content Editor toolbar (or Ctrl+S). The extension's bridge layer has already fired all four change-notification strategies, so the Save button is active. Publish the item as normal when ready.

Managing Repeat Items

Adding, reordering, and removing items

Adding a new item

e.g. adding a fourth plan card or a new meeting

At the bottom of the repeat group list, click + Add [Item]. A new group appears with blank fields. Fill in the fields and click Apply Changes. The extension appends the new item to the HTML after all existing items.

Removing an item

e.g. removing an expired plan or past meeting

Click the × button on a group header to mark it for removal. The group collapses and is excluded when you click Apply Changes. The item is removed from the HTML on save.

For the Zoom Meetings widget, past meetings are flagged visually in the sidebar using the data-end attribute — items where the end date is in the past are shown with a faded style to indicate they can safely be removed.

Updates

Handling extension and registry updates

There are two types of updates: registry updates (new widgets or field changes) and extension updates (new extension code).

Registry updates — automatic, no reinstall

New widgets, modified fields, updated selectors

The registry is fetched fresh every time the sidebar opens. If the version string in the registry changes, a gold badge appears on the extension icon and an in-sidebar banner displays with a note about the update. No action is required from the user — the new registry is already active.

Extension updates (new zip) follow a four-step process:

Step 1

Badge appears

A gold badge on the extension icon and a banner in the sidebar indicate a new extension version is available, with a download link.

Step 2

Download ZIP

Visit the Install page to download the latest .zip. Extract it to a folder.

Step 3

Remove old extension

Go to chrome://extensions and remove the old version of Sitecore Widget Editor before loading the new one.

Step 4

Load unpacked

Click "Load unpacked" and select the newly extracted folder. The extension reloads with the new version — settings and recents are preserved in Chrome storage.

Why not the Chrome Web Store? — The extension is an internal tool that calls a Sitecore-specific SSC API with a hardcoded API key scoped to a specific Sitecore instance. Publishing it publicly would expose that key and would serve no purpose for users outside that environment. Load-unpacked distribution is intentional.
Troubleshooting

Common issues

"No widgets detected" — but a widget is on the page

This means the widget's root CSS class isn't in the registry, or the item selected in the Content Editor tree isn't the item that contains the Rich Text field with the widget HTML.

Check that you're on the correct item (the one whose Rich Text field holds the widget markup) and that the widget's class is registered in widget-registry.json.

Apply Changes completes but Sitecore Save button stays inactive

The bridge fires four change-notification strategies, but some Sitecore configurations may not respond to all of them. Try clicking inside the Rich Text field in the Content Editor after applying, then saving manually. If the issue is persistent, the bridge strategies may need tuning for your specific Sitecore version.

The sidebar shows a startup error or blank white panel

This usually means the extension loaded from a folder where one of the bundled files is missing or corrupted. Re-extract the ZIP to a clean folder and reload via chrome://extensions. The most common cause is a partial extraction or a moved folder after loading.

Rich text fields show as plain text inputs

This means quill.js failed to load. Check that the extension folder contains quill.js and quill.snow.css at the root. If Quill fails to initialize, the field falls back to a plain text input so data is not lost — but formatting is unavailable.
