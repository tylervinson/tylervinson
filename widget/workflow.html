Widget Workflow

Complete walkthrough from development to production maintenance

Extension Overview Widget Workflow Markup Generator
Phase 1

Design & Development

Begin by designing and building the widget in your local development environment. Focus on creating clean, semantic HTML with proper structure. At this stage, no special classes or data attributes are needed.

Write standard HTML using content elements like headings, paragraphs, links, and images. The markup generator will handle the rest.

  • Design: Create mockups and finalize visual design
  • Build: Write HTML, CSS, and any JavaScript functionality
  • Test: Verify widget works in development environment
  • Prepare: Clean up markup, remove unnecessary classes
Development Screenshot
Phase 2

Markup Generation

Use the HTML Auto-Annotator to transform your clean HTML into Sitecore-ready markup. The generator automatically detects content elements and adds all necessary data attributes.

Provide the widget class name, display name, and optional description. The generator wraps your HTML with .editable-widget class and adds data-* attributes to all editable content elements.

  • Open Generator: Navigate to the HTML Auto-Annotator tool
  • Configure: Enter widget class, display name, description
  • Paste HTML: Input your clean widget HTML
  • Generate: Click to process and auto-annotate
  • Copy Output: Get fully annotated HTML ready for Sitecore
Generator Interface Screenshot
Phase 3

Sitecore Integration

Deploy the annotated HTML to Sitecore. The markup now contains all necessary metadata for the extension to automatically discover and manage the widget.

No manual registration or code updates required. The extension will immediately detect any widget with the .editable-widget class when the page loads.

  • Copy HTML: Take annotated markup from generator
  • Add to Sitecore: Paste into appropriate component or template
  • Include CSS/JS: Add any associated stylesheets and scripts
  • Publish: Deploy changes to staging or production
  • Verify: Check widget renders correctly on page
Sitecore CMS Screenshot
Phase 4

Content Management

Once deployed, the Chrome extension automatically discovers the widget and provides a visual editing interface. Content editors can make changes directly through the browser without touching code or accessing the CMS.

The extension reads all data-* attributes, builds an appropriate form interface, and writes changes back to both the attributes and visible page content in real-time.

  • Open Page: Navigate to page containing the widget
  • Launch Extension: Click extension icon to open sidepanel
  • Select Widget: Choose from auto-discovered widgets
  • Edit Content: Update text, links, images through form interface
  • Apply Changes: Save updates back to page
  • Notify Sitecore: Extension triggers CMS to persist changes
Extension Sidepanel Screenshot
Phase 5

Ongoing Maintenance

The extension requires no updates when new widgets are added. Simply follow the same process: generate annotated HTML, add to Sitecore, and the extension automatically detects it.

For existing widgets, content updates happen in real-time through the extension. Structural changes require regenerating the HTML through the markup generator.

  • Content Updates: Use extension for text, link, and image changes
  • Structural Changes: Regenerate HTML if adding/removing elements
  • New Widgets: Run through markup generator, no extension updates
  • Exclusions: Add .exclude classes to prevent editing specific elements
  • Team Access: All team members use same extension version
