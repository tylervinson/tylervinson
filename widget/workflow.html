Widget Workflow

From development to day-to-day content management

Extension Overview Widget Workflow Technical Specs
Phase 1

Design & Development

Build the widget as you normally would — clean HTML, CSS, and JavaScript. No special attributes or classes are needed at this stage. Focus on getting the markup and visual output right.

Once the widget is working correctly, decide which elements content authors will need to edit. These become the field definitions in Phase 2.

  • Design: Finalize mockups and visual design
  • Build: Write HTML, CSS, and JavaScript
  • Test: Verify the widget renders and behaves correctly
  • Identify fields: Note which text nodes and attributes need to be editable
Phase 2

Registry Registration

Open widget-registry.json and add a definition for the new widget. The definition specifies the widget's root CSS class, its display name, and the full set of editable fields — each with a CSS selector, a label for the sidebar form, and any data attribute sync rules.

If the widget has repeating items (cards, plan buttons) define a repeat block with the selector for each item. Standalone fields like headings go in standaloneFields. Fields that update a separate comparison table row use gridSelector. No Sitecore item IDs or field IDs are needed — the extension finds those automatically from the page.

  • Open registry: Edit widget-registry.json on the server
  • Add widget entry: Class name, display name, field definitions
  • Define fields: Selector, label, type, sync attribute if needed
  • Push to server: Update the version number and deploy
Phase 3

Sitecore Deployment

Deploy the widget HTML to Sitecore as you normally would. No special markup is required — the registry selectors do the mapping.

  • Paste widget HTML into Sitecore component
  • Include CSS and JS assets as usual
  • Publish to staging or production
  • Verify widget renders correctly on the page
Phase 4

Content Management

Install the extension, open the Content Editor, navigate to a component, and click the icon. The sidebar reads the current item via SSC API and presents the editor form immediately — no widget selection step needed.

  • Install or update the extension ZIP
  • Open Sitecore Content Editor
  • Navigate to a component in the tree
  • Open the sidebar via the extension icon
  • Edit fields and click Apply Changes
  • Save in Sitecore
Phase 5

Ongoing Maintenance

Content updates happen entirely through the sidebar. Structural changes require only a registry edit and server push — no extension rebuild.

  • Content updates via sidebar form
  • New fields or widgets via registry edit
  • Push updated JSON to server
  • Badge icon notifies authors of registry update
  • No extension reinstall required for registry changes
How to Use

Using the Extension

1

Open the Content Editor

Navigate to Sitecore's Content Editor and find the component you want to edit in the content tree on the left.

2

Select the Component

Click the component item in the tree. The Quick Info panel on the right will show the item ID, name, and path — the extension reads this automatically.

3

Open the Extension

Click the extension icon in your Chrome toolbar to open the sidebar panel. It will detect the current component and populate the edit form immediately.

4

Edit the Fields

Update any field values in the form. Repeating items like plan cards or stat blocks are grouped individually. Standalone fields like headings appear at the top.

5

Apply Changes

Click Apply Changes. The extension silently opens the HTML editor, writes the updated markup, and closes it — no manual WYSIWYG interaction needed.

6

Save in Sitecore

Click Save in the Content Editor toolbar. Your changes are now live in Sitecore.
