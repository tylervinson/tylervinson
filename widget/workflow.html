Widget Workflow

Complete walkthrough from development to production maintenance

Extension Overview Widget Workflow Code Generator
Phase 1

Design & Development

Build the widget as you normally would — clean HTML, CSS, and any JavaScript. No special attributes are needed at this stage. Focus on getting the markup and visual output right.

Once the widget looks and behaves correctly, identify the elements content authors will need to edit. These are the fields you'll mark in Phase 2.

  • Design: Create mockups and finalize visual design
  • Build: Write HTML, CSS, and any JavaScript
  • Test: Verify the widget renders correctly
  • Identify fields: Note which elements need to be editable
Phase 2

Markup Generation

Open the Code Generator and choose your approach. Use Generate Widget if you want explicit control — add item-* classes to your field elements (e.g. item-name, item-price) and the generator reads them automatically. Use Interactive Mode if you'd rather not think about naming conventions — just add editable to any element and the generator walks you through naming each field inline.

Either way: enter the widget class and display name, paste your HTML, and click Generate. The generator annotates the HTML, registers the widget, pushes the updated registry to GitHub Pages, and downloads the extension ZIP — all in one step.

  • Mark fields: Add item-* classes or editable to editable elements
  • Configure: Enter widget class, display name, and description
  • Paste HTML: Drop your widget markup into the generator
  • Generate: One click produces annotated HTML + extension ZIP
  • Copy output: Paste the annotated HTML into Sitecore
Phase 3

Sitecore Integration

Deploy the annotated HTML to Sitecore. The markup now contains all necessary metadata for the extension to automatically discover and edit the widget.

  • Copy annotated HTML from generator output
  • Paste into Sitecore component
  • Include CSS and JS assets
  • Publish to staging or production
  • Verify widget renders correctly
Phase 4

Content Management

Install the generated extension ZIP. The sidepanel detects the widget automatically and presents a grouped editor — items and grid sections match the widget's structure exactly.

  • Install extension from generated ZIP
  • Navigate to page with widget
  • Click extension icon to open sidebar
  • Edit fields and click Apply Changes
  • Save in Sitecore Experience Editor
Phase 5

Ongoing Maintenance

When you generate a new or updated widget, the registry is pushed to GitHub Pages automatically. Content authors see an update banner in the sidebar and can download the new version directly.

  • Content updates via extension sidebar
  • Structural changes via generator
  • Registry auto-pushed to GitHub Pages
  • Update banner notifies content authors
  • Team always installs from same source
