OVERVIEW

Visual Editing for Data-Driven Sitecore Widgets

Editing data-driven widgets in Sitecore traditionally requires manual manipulation of HTML attributes and inline data structures. Content editors must dig through source code to modify insurance plan details, stats cards, or comparison tables.

This Chrome extension bridges that gap. It scans the page for registered widgets, reads their data from data-* attributes, and presents a clean grouped form in a persistent sidebar. Changes sync instantly back to the DOM and notify Sitecore's Experience Editor to enable the save button.

The extension is generated automatically — paste your widget HTML into the Code Generator, mark editable fields, and download a ready-to-install ZIP that includes the full extension plus your widget's editor.

CAPABILITIES

Three Widget Modes, One Extension

The extension supports three widget structures — items (repeating cards or rows), grid (comparison tables with plans and fields), and composite (both together). The Code Generator detects which mode applies automatically and generates the right editor UI for each.

A registry pushed to GitHub Pages drives everything. When you generate a new widget, the registry updates automatically — content authors with the extension installed see an update banner and can install the new version in one click.

Core Features

Automatic Detection

Scans the page DOM for registered widget classes with mutation observer support for dynamically loaded content

Three Widget Modes

Items (repeating elements), Grid (data-plan/data-field comparison tables), and Composite (both in one widget) — all handled automatically

Grouped Editor UI

Sidepanel forms are grouped to match the widget's structure — plan cards separate from comparison grid rows, with inferred field types

DOM Synchronization

Updates both data attributes and visible text content simultaneously to keep the widget consistent

Sitecore Bridge

Multi-strategy change notification triggers TinyMCE sync, field-level change tracking, and chrome events to enable the save button

GitHub-Backed Registry

Widget registry is auto-committed to GitHub Pages on every generation — content authors always have access to the latest widget definitions

ARCHITECTURE

Generator → Registry → Extension

The Code Generator is the source of truth. When you generate a widget it annotates your HTML, registers the widget definition, commits the updated widget-registry.json to GitHub Pages, and downloads an installable extension ZIP — all in one step.

The extension reads the registry on startup and checks for updates periodically. When a new version is available, a banner appears in the sidepanel with a direct download link. The SitecoreBridge module handles the Sitecore-specific challenge of making DOM changes visible to TinyMCE's internal content model so the save button activates correctly.
