OVERVIEW

Visual Editing for Data-Driven Sitecore Widgets

Editing data-driven widgets in Sitecore traditionally requires navigating source code to manually update inline values, data attributes, and HTML structure. Content editors shouldn't need to touch markup to change a stat, a plan price, or a card label.

This Chrome extension solves that. It opens as a persistent sidebar inside the Content Editor, reads field values directly via the Sitecore SSC API, and presents a clean grouped form. Hit Apply Changes and the extension silently automates the HTML editor — opening it, writing the updated markup, and closing it — no manual WYSIWYG interaction required.

Widget support is driven entirely by a central widget-registry.json hosted on the server. Adding or updating a widget requires only editing that file — no changes to the extension itself.

HOW IT WORKS

Registry-Driven. No Code Changes Required.

When the sidebar opens, it fetches the latest registry from the server. It then reads the currently selected item's ID directly from the Content Editor's Quick Info panel and queries the Sitecore SSC API for all rich text fields on that item. Any field whose HTML contains a registered widget class is detected automatically — no item IDs needed in the registry.

When a new widget needs to be supported, a developer adds its CSS class and field definitions to the registry JSON and pushes it. Content authors open the sidebar the next time and the new widget is already there — no extension update required. If the registry version changes, a notification badge appears on the extension icon.

Core Capabilities

Content Editor Integration

Works inside Sitecore's Content Editor. Reads the selected item from the Quick Info panel and fetches field data via the SSC API — no WYSIWYG required to read.

Registry-Driven Forms

The sidebar form is generated entirely from field definitions in the registry — labels, input types, and grouping all come from the JSON. No item IDs or field IDs needed.

Grouped Editor UI

Repeating items (cards, plans) are presented as collapsible groups. Standalone fields appear above. Structure mirrors the widget itself.

Automated WYSIWYG Writes

On Apply, the extension programmatically opens the HTML editor, writes the updated markup, and closes it — the whole cycle completes silently in under a second.

Navigation Awareness

Watches the Content Editor tree selection. When a different component is selected, the form clears and rescans automatically — no manual refresh needed.

Live Update Notifications

Background service worker checks registry version periodically. A badge on the extension icon signals when a new version is available.

Architecture

widget-registry.json

Hosted on the server. Defines every widget — class name, field selectors, labels, repeat structure, and data attribute sync rules. No item IDs required. Single source of truth.

Sidepanel

Fetches the registry on open. Reads the selected item ID from Quick Info, detects widgets via SSC API, builds the editor form, and passes full widget definitions into every message sent to the content script.

Content Script

Handles all Sitecore interaction. Fetches field HTML via SSC API, parses it with DOMParser, and on save automates the HTML editor open/write/close cycle.

Background Worker

Routes messages between the sidepanel and content script. Checks for registry version updates every 4 hours. Sets the extension badge when a new version is detected.

SSC API

Sitecore's aggregate content API at /sitecore/api/ssc/aggregate/content/Items. Used to read all rich text fields for the current item without opening any editor modal.

Adding a Widget

Edit widget-registry.json, add a widget definition with class, fields, and selectors, push to server. No item IDs, no extension rebuild, no reinstall needed.
