Process Comparison

Manual vs Automated Integration

Manual integration requires careful coordination across multiple files with exact selector matching and proper data flow. The automated process reduces this to markup annotation and basic configuration.

Manual Integration
  • Add data-* attributes to HTML
  • Update WIDGET_REGISTRY in content-script.js
  • Write getData() function with selectors
  • Write updateData() function
  • Update router functions
  • Create editor UI in sidepanel.js
  • Write form generation logic
  • Wire event handlers
  • Update manifest.json version
  • Test and debug selector mismatches
  • Package and distribute files
Automated Integration
  • Annotate HTML with dynamic-* classes
  • Upload current extension ZIP
  • Input widget metadata
  • Paste annotated HTML
  • Click Generate
  • Download updated extension
Generator Functions

Automated Implementation

The generator performs structure analysis, HTML processing, code injection, editor generation, and complete file packaging. Each function is tested and produces consistent code across all widgets.

Automated Functions

Structure Analysis

Detects widget pattern, identifies field types, determines code generation strategy.

HTML Processing

Converts dynamic-* classes to numbered data-* attributes, replaces classes.

Code Injection

Adds registry entry, injects router cases, appends widget functions.

Editor Generation

File Packaging

State Preservation

Markup Specification

Available Annotations

Use dynamic-object to mark repeating elements and dynamic-{type} classes to mark editable fields. The generator converts these annotations into proper data attributes and generates corresponding editor inputs.

Dynamic Classes

dynamic-object

Marks repeating items like cards, list items, or table rows.

dynamic-text

Plain text fields for names, titles, labels.

dynamic-number

Numeric values for prices, quantities, percentages.

dynamic-email

Email addresses with validation.

dynamic-phone

Phone numbers with tel input type.

dynamic-url

Web addresses with URL validation.

dynamic-textarea

Multi-line text for descriptions, paragraphs.

dynamic-date

Date values with calendar picker.

dynamic-checkbox

Boolean values for toggles, flags.

Integration Steps

Implementation Sequence

After widget design and development are complete, integration follows this sequence: annotate markup, upload current extension, configure metadata, generate code, and deploy.

Step Sequence

1. Annotate Markup

Add dynamic-object and dynamic-{type} classes to widget HTML.

2. Upload Extension

Load current extension ZIP to preserve existing widgets.

3. Configure Widget

Provide class name, display name, description, and paste HTML.

4. Generate

Generator processes HTML and injects code into all files.

5. Review Output

Verify processed HTML for use in Sitecore.

6. Deploy

Extract ZIP, reload extension, test widget.
