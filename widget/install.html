Browser

Google Chrome — any recent version. The extension uses Manifest V3 and the Chrome Side Panel API.

Environment

Sitecore Content Editor — the extension works in the Content Editor only. Experience Editor is not supported.

Distribution

Load unpacked — not available on the Chrome Web Store. Developer mode must be enabled in Chrome.

Installation Steps
1

Download and extract the ZIP

Click the download button above. Save the ZIP somewhere you won't accidentally move it — Chrome references the folder by its path on disk after loading. A stable location like ~/Documents/chrome-extensions/sitecore-widget-editor/ works well.

Extract the ZIP. The extracted folder should contain manifest.json at its root alongside the other extension files. Do not rename or restructure the folder.

⚠ Do not move the folder after loading it in Chrome. Chrome stores the path and will show an error if the folder is relocated. Re-load it from the new path if you ever move it.

2

Open Chrome Extensions and enable Developer mode

In Chrome, navigate to chrome://extensions in the address bar (or open the Chrome menu → More tools → Extensions).

In the top-right corner of the Extensions page, find the Developer mode toggle and turn it on. Three new buttons will appear: "Load unpacked", "Pack extension", and "Update".

Developer mode must remain enabled for the extension to keep working. Chrome will occasionally remind you that developer extensions are active — this is normal for load-unpacked extensions.

3

Load the unpacked extension folder

Click Load unpacked. A system file picker opens. Navigate to and select the extracted extension folder — the one that has manifest.json at its root. Click "Select Folder" (or "Open" on macOS).

The extension will appear in your extensions list with the name Sitecore Widget Editor and version 4.4.6. Its icon will also appear in the Chrome toolbar.

If you see an error like "Manifest file is missing or unreadable" it means you selected the wrong folder level — open one level deeper until you see manifest.json inside.

4

Pin the extension to your toolbar

The extension icon may be hidden in the extensions overflow menu (the puzzle-piece icon ⧉ in the toolbar). Click it to see all installed extensions, find Sitecore Widget Editor, and click the pin icon to make it permanently visible in the toolbar.

Once pinned, clicking the icon on any Sitecore Content Editor page will open the sidebar. Installation is complete.

5

Open the sidebar in the Content Editor

Navigate to Sitecore's Content Editor and select an item in the tree that contains a registered widget in a Rich Text field. Click the Sitecore Widget Editor icon in the toolbar.

The sidebar opens on the right side of the browser window. It shows the dashboard first — with content search, recently viewed items, and checked-out items. When a widget is detected on the selected item, the editor form loads automatically.

Updating the Extension
When a new version is available, a gold ! badge appears on the extension icon and a download banner shows inside the sidebar. Registry updates (new widgets or field changes) apply automatically — no reinstall needed. Extension code updates require a manual reinstall.
Step 1

Badge appears

Gold ! badge on the icon and a banner inside the sidebar with a download link.

Step 2

Download new ZIP

Click the link in the banner or download from this page. Extract to a folder.

Step 3

Remove old version

Go to chrome://extensions, find Sitecore Widget Editor, and click Remove.

Step 4

Load unpacked

Click "Load unpacked" and select the new folder. Done — recents and settings are preserved.

Common Issues
"Manifest file is missing or unreadable"
You selected the wrong folder level. Open one level deeper — the folder you select must contain manifest.json directly inside it, not inside a subfolder.
Extension icon doesn't appear in toolbar
Click the puzzle-piece (⧉) icon in the Chrome toolbar. Find "Sitecore Widget Editor" in the list and click the pin icon next to it to make it permanently visible.
"Extension could not be loaded" after moving files
Chrome stores the path to the extension folder. If you moved it, remove the broken extension from chrome://extensions and re-load it from its new location using "Load unpacked".
Sidebar opens but shows nothing / blank white panel
A bundled file is likely missing or corrupt. Re-extract the ZIP to a clean folder and reload. Check that the extracted folder has all 11 files listed in Step 1.
Chrome shows "This extension is not from the Chrome Web Store"
This is expected. The extension is an internal tool distributed as a load-unpacked extension. Click Keep it or Enable anyway when Chrome shows this warning. It does not indicate a security problem — it means the extension is not from the Chrome Web Store, which is correct.
