Installation Guide

1

Download the Extension

Click the "Download Extension" button at the top of this page. This will download a ZIP file containing the complete extension code.

Note: Keep this file in an easy-to-find location like your Downloads folder—you'll need the unzipped folder in Step 3.
2

Unzip the File

Extract the ZIP file. On Mac, double-click the ZIP. On Windows, right-click and select "Extract All." You should now have a folder named something like "Formstack-Template-Maker."

3

Open Chrome Extensions Page

In Google Chrome, type chrome://extensions in the address bar and press Enter. Alternatively, open the Chrome menu (⋯), select "More Tools," then "Extensions."

4

Enable Developer Mode

In the top right corner of the Extensions page, toggle the switch labeled "Developer mode" to ON. You should now see a blue "Load unpacked" button.

Important: This is a standard Chrome feature—enabling Developer mode is safe and won't affect your browser.
5

Load the Extension

Click the "Load unpacked" button. A file picker will open. Navigate to and select the unzipped extension folder (the one that contains manifest.json), then click "Select Folder."

6

Pin the Extension

The extension should now appear in your Chrome toolbar. Click the puzzle icon in the top right, find "Formstack Documents HTML Theme Generator," and click the pin icon to keep it visible.

Done! You can now use the extension whenever you're working in Formstack Documents.

How to Generate an HTML Template

Generation Interface Screenshot
1

Prepare Your Document

In Formstack Documents, create a new document and import your Formstack form as the data source using the Document Builder. The Initial Template should be visible.

Need help? See the HTML Generation Workflow page for detailed setup instructions.
2

Open the Extension

With the Formstack Documents page open, click the extension icon in your Chrome toolbar. A popup will appear with a "Scan Current Page" button.

3

Scan the Page

Click "Scan Current Page." The extension will analyze the Initial Template on the page, detect all form fields, and generate the HTML and test data.

Typical scan time: Less than 500 milliseconds.
4

Copy the HTML

Once the generation completes, you'll see the HTML in the first text area. Click the "Copy" button to copy the entire HTML to your clipboard.

5

Paste into Your Document

In your Formstack Document, open the "Advanced HTML Editor." Paste the HTML from the extension (it will replace the existing template). Save your document.

6

Test with JSON Data

Copy the JSON test data from the extension. In your document, open the "Test" tab, scroll to the bottom, paste the JSON, and click "Test Document" to preview how your document will render.

Pro tip: Save the JSON somewhere for future testing. You can use the same JSON to test multiple versions of your HTML.

Customizing Your HTML

The generated HTML is production-ready, but you can customize it to match your brand or add specific styling and functionality.

Adding CSS Classes for Styling

The generated HTML includes semantic class names like .form-section, .form-field, and .field-label. Add your CSS directly in the HTML editor or create a <style> block to customize fonts, colors, spacing, and layout.

Modifying Field Layout

You can adjust how fields are grouped or arranged. The generated HTML preserves the original form structure, but you can add CSS Grid or Flexbox rules to create multi-column layouts, adjust spacing, or reorder fields visually.

Adding Custom JavaScript

For dynamic functionality like field validation, calculated fields, or conditional display, add JavaScript in a <script> block. The generated HTML structure makes it easy to target elements with standard DOM selectors.

Testing Your Changes

After making edits, save your document and use the same JSON test data to preview your changes. The Test tab will show you exactly how your customizations render with real form data.

Frequently Asked Questions

What if my form changes after I generate the HTML?

Simply click "Scan Current Page" again to regenerate the HTML with the updated fields. You can update your document's HTML Editor with the new version. Any custom CSS or JavaScript you added will remain, as long as the field class names haven't changed significantly.

Can I use the same HTML for multiple Formstack Documents?

If your forms have identical structure, you can use the same HTML as a starting point. However, if field names or structure differ between forms, you'll get better results by generating a fresh HTML template for each document.

Does the extension send my form data anywhere?

No. The extension works entirely in your browser. It only analyzes the DOM of the page you're viewing. No data is sent to servers, and no information is collected or stored. Your form data remains completely private.

What if the extension doesn't find my form?

Make sure you're viewing the Formstack Documents page with the Initial Template visible. The extension scans the page DOM directly. If the template isn't rendering, the extension won't be able to detect it. Try refreshing the page or checking that your form imported successfully.

Can I edit the HTML and JSON after copying them?

Yes. Both the HTML and JSON are plain text. You can edit them in the extension text areas before copying, or paste them into any text editor for more advanced editing before pasting into Formstack Documents.

Is there a limit to how many fields the extension can handle?

No hard limit. The extension scans the page DOM and generates HTML for all fields present. Very large forms (200+ fields) may take slightly longer to scan, but should still complete in under a second.

Troubleshooting

Extension icon doesn't appear in toolbar

Make sure you've loaded the extension (Step 5 of Installation). If it's loaded but not visible, click the puzzle icon (extensions menu) and pin the extension. If it doesn't appear at all, reload the page and try again.

"Cannot load extension" error during installation

This usually means the folder you selected doesn't contain a valid manifest.json file. Make sure you selected the unzipped folder itself, not a subfolder inside it. The folder should contain manifest.json at the root level.

"Scan Current Page" button doesn't work

The extension only works on Formstack Documents pages. Make sure you're viewing a document with an Initial Template visible. If you're on the right page, try refreshing and clicking the button again. You should see a message indicating the scan is in progress.

Generated HTML looks incomplete or missing fields

This can happen if the Initial Template isn't fully rendered when you click "Scan." Try waiting a few seconds for the page to load completely, then refresh and scan again. Very large forms may need a moment to render all fields.

Test data doesn't match my field types

The extension generates basic test data for each field type (email, phone, text, etc.). If you need specific test values, you can edit the JSON before pasting it into the Test tab. Update any values that don't match your needs.

Copy button doesn't work

The copy functionality uses your browser's clipboard API, which requires user permission. Make sure you're clicking the copy button, not trying to select and copy manually. If it still doesn't work, you can manually select all the text and copy it (Ctrl+C or Cmd+C).
