Widget HTML Markup Specifications

Requirements for Code Generator Compatibility

Extension Overview Code Generator Markup Specs

Required HTML Structure

Your widget HTML stores all editable data in data-* attributes. Visible content can be generated from these attributes by your widget's JavaScript.

Three Requirements
  1. Outer container: Unique CSS class identifying the widget type (e.g., team-member-bio)
  2. Item identifiers: Each repeating item needs data-{itemtype}-id (e.g., data-member-id="1")
  3. Data attributes: All editable fields as data-* attributes on the item element

Minimal Structure (Data Only)

<div class="team-member-bio">
  <div data-member-id="1"
       data-name="John Smith"
       data-title="Senior Developer"
       data-email="john@company.com">
    <!-- Your widget's JavaScript renders this -->
  </div>
</div>

With Visible Content (Optional)

If your widget already renders visible content, the extension will update both the data-* attributes AND the visible elements to keep them in sync:

<div class="team-member-bio">
  <div data-member-id="1"
       data-name="John Smith"
       data-title="Senior Developer"
       data-email="john@company.com">
    <!-- Extension updates these when you save -->
    <h3 class="member-name">John Smith</h3>
    <p class="member-title">Senior Developer</p>
    <p class="member-email">john@company.com</p>
  </div>
</div>
Important: The source of truth is the data-* attributes. Visible content is just for display and will be overwritten when the extension saves changes.

Naming Conventions

Widget class: lowercase-with-hyphens (e.g., team-member-bio, pricing-calculator)

Item identifier: data-{singular-itemtype}-id (e.g., data-member-id, data-plan-id)

Field attributes: data-{field-name} (e.g., data-name, data-email, data-bio)

Good Examples

team-member-bio
insurance-comparison
testimonial-carousel
pricing-calculator

Avoid

widget
component
box
content

Auto-Detected Input Types

The generator creates appropriate form inputs based on field names:

Field Name Contains Input Type
email Email input
phone Tel input
url, link URL input
bio, description Textarea
anything else Text input

Complete Example

<div class="team-member-bio">
  
  <div data-member-id="1"
       data-name="Sarah Johnson"
       data-title="VP of Engineering"
       data-email="sarah@company.com"
       data-phone="555-0001"
       data-bio="Sarah has over 15 years of experience.">
    <!-- Optional: visible content your widget renders -->
    <h3 class="member-name">Sarah Johnson</h3>
    <p class="member-title">VP of Engineering</p>
    <p class="member-email">sarah@company.com</p>
  </div>

  <div data-member-id="2"
       data-name="Michael Chen"
       data-title="Lead Designer"
       data-email="michael@company.com"
       data-phone="555-0002"
       data-bio="Michael specializes in UX design.">
    <h3 class="member-name">Michael Chen</h3>
    <p class="member-title">Lead Designer</p>
    <p class="member-email">michael@company.com</p>
  </div>

</div>

Generator detects:

  • Widget class: team-member-bio
  • Item type: member (from data-member-id)
  • Fields: name (text), title (text), email (email input), phone (tel input), bio (textarea)

Updating Visible Content

The generated code will update visible elements IF they follow the naming pattern .{itemtype}-{field-name}:

data-name → .member-name
data-email → .member-email
data-title → .member-title

If your CSS classes don't match this pattern, you'll need to manually update the querySelector selectors in the generated update function.

Auto-works

<div data-name="John">
  <h3 class="member-name">
    John
  </h3>
</div>

Needs Manual Update

<div data-name="John">
  <h3 class="person-fullname">
    John
  </h3>
</div>

Pre-Generator Checklist

  1. ✓ Widget has unique class name
  2. ✓ Items have data-{itemtype}-id
  3. ✓ All editable fields are data-* attributes
  4. ✓ Naming uses lowercase-with-hyphens
  5. ✓ Sample HTML includes 2+ items with all fields
  6. ✓ (Optional) Visible content classes follow .{itemtype}-{field} pattern
